# Sales Enablement

Build assets that help reps have better conversations and close more deals.

## Asset Types

**Pitch Decks**: 10–12 slide story arc — Problem → Solution → Proof → ROI → Next Steps

**One-Pagers**: Scannable leave-behinds for post-meeting follow-up or champion enablement

**Objection Handling Docs**: Quick-reference table mapping objection → response → proof point

**Demo Scripts & Talk Tracks**: Scene-by-scene walkthrough with timing and interaction prompts

**ROI Calculators**: Input/output models showing time saved, cost reduction, and payback period

**Sales Playbooks**: Buyer profiles, qualification criteria, discovery questions, competitive positioning

**Buyer Persona Cards**: Role-specific goals, pains, objections, and messaging angles by title

**Proposal Templates**: Executive summary → implementation plan → pricing → next steps

## Required Inputs

1. What asset is needed?
2. Who uses it (AEs, SDRs, champions, prospects)?
3. Value proposition and key differentiators
4. Target persona (title, department, main concern)
5. Top 3 objections you hear most

## Objection Handling Structure

For each objection:
- Acknowledge (validate the concern)
- Reframe (shift perspective)
- Prove (specific proof point or customer story)
- Advance (suggest next step)

## Integration

- product-marketing: foundational context for all assets
- customer-research: real objections from real buyers
- copywriting: asset copy quality
- revops: where assets fit in the CRM workflow
